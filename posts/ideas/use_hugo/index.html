如何使用hugo搭建个人博客（以eureka主题为例）？

2024-02-12
4分钟阅读时长
blog
hugo

1. 引言

本文介绍如何使用hugo这个静态网站生成器来生成自己的个人博客。并使用Github Pages托管网站使网页可以被在公网访问。

以下所提内容并不深入，只是搭建自己博客过程的记录。

2. 安装hugo及创建个人博客

2.1 安装hugo

hugo官方文档

我建议直接安装预构建版本，因为只是使用，并不在其基础上做出任何改进。

linux的x86机器安装amd64的版本，windows同理。

这里该有个图片，但我图床还没弄好

把压缩包解压后会出现一个名为hugo的可执行文件，将该文件所在路径加入到环境变量PATH中，(linux的话也就是执行export PATH=/path/to/hugo:$PATH)。然后运行

hugo version

正确输出即为配置成功。

2.2 创建个人博客

找一个空闲目录，执行下面操作：

hugo new site site # site是你网站的名字，随便起名

然后就是安装一个hugo的主题了，这里我们选择eureka，个人觉得这个主题比较简洁。按作者文档介绍中所述切换到对应Release版本。例：

cd site
git init
git submodule add https://github.com/wangchucheng/hugo-eureka.git themes/eureka
cd themes/eureka
git checkout 749f68c # 这里的749f68c对应Release的v0.9.3版本
cd -

随后删除当前根目录下的config文件，现在也许叫hugo.toml。将eureka主题提供的config与content复制到当前项目的根目录下。例：

rm hugo.toml
cp -r themes/eureka/exampleSite/* ./

随后修改config/_default/config.yaml文件，将theme: eureka取消注释并将theme: wangchucheng.com/hugo-eureka注释。

然后我们就可以使用hugo提供的webserver在本地预览博客啦！运行:

hugo server

想要更改博客内容就只需要更改config和content文件夹就可以，这个根据作者文档来修改就可以，介绍比较详细。

3. 使用Github Pages存放网站

现在我们已经可以本地预览博客内容了，但是我们肯定不是为了只搞一个自己看的东西，让我们把他放到公网上大家都访问吧！

3.1 打包部署

在项目的根目录下运行hugo命名，即可生成public文件夹，文件夹内包含各种静态资源(html,css,js)。常规部署网站是使用Nginx指向静态资源根目录下的index.html，所以如果使用Nginx部署在自己的服务器上，那就正常部署就好，我就不过多赘述了，毕竟如何使用Nginx也不是本文的主题。我们使用Github Pages服务，让Github来帮我们部署静态资源。

首先创建一个名称如username.github.io的github仓库（username为你的用户名），来使用github pages。创建好后将项目目录下public文件建立与该仓库的连接，并将文件夹下内容推送到仓库里。然后访问https://username.github.io/即可访问该网页。

3.2 使用脚本自动化打包部署

熟悉如何部署后，就可以使用脚本来代替我们完成这个更新过程了。

我们分别把项目主体(hugo eureka)放在一个github仓库里，再按上面所说将public文件夹放在github pages的仓库里。然后在hugo eureka下创建脚本update.sh，内容示例如下：

# 更新本仓库

git pull
git add .
git commit -m "new version"
git push

echo "Update finish"

# 用于发布新网站至Github Pages
echo "Currently path ${PWD}."

echo "Running 'hugo' to build folder 'public'."

hugo

echo "Go to the 'public' folder"

cd public

echo "Update new content to username.github.io"

git pull
git add .
git commit -m "new site"
git push

cd -

每次更新内容后运行此脚本即可更新内容到Github Pages。

还有一种是使用Github Action的方案，大致叙述一下：

把hugo-eureka存在github的一个仓库，通过设置action的触发条件，比如每次push都触发一次，将hugo-eureka的仓库更新后，通过设置.github/workflow下的文件，在github的服务器上运行hugo打包部署，然后将打包后的文件传到github pages的仓库。这样就实现了自动部署，体现在上面的脚本上就是删除了# 用于发布新网站至Github Pages后的内容，将这部分内容由github代为执行。这样就可以不再本机上安装hugo了，也就是可以更换电脑依旧能够写博客。

之所以我不用Github Action，是因为我觉得在本地仓库推送到Pages仓库可定制化强一些，不至于让Pages仓库只能有hugo生成的文件内容，可以在Pages仓库存放一些自己想放的东西。

4. 关于icon的bug

我在使用过程中发现了一个关于icon的bug，比较影响使用体验。就是想要在除en语言的其它语言页面，想要添加其它icon，是添加不上的。这个icon必须在en的页面先使用过才可以加载出来。所以我在en页面下添加了一个author介绍页，就是这个。我把所有需要用到的icon都在这个页面先加载一下，再拿到别的地方用。

5. 结语

GoodBye
